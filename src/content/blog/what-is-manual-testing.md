**Manual testing** means a human tester checks the software by using it — clicking, typing, and observing — without running automated scripts. You act like a user and verify that the app behaves correctly.

## Why manual testing matters

- **Exploration:** You can try unexpected inputs, odd workflows, and edge cases that scripts might not cover.
- **UX and visuals:** You notice layout issues, confusing text, and usability problems that automation often misses.
- **Early stages:** When the product changes often, manual testing is fast to adapt; automation takes time to build and maintain.

## Manual vs automated testing

| Manual testing | Automated testing |
|----------------|-------------------|
| A person runs the steps and checks results | Scripts run the steps and compare results |
| Flexible, good for exploration and UX | Repeatable, good for regressions and CI |
| Slower for large regression suites | Fast for running the same checks many times |

Both are useful. Manual testing is where many testers start, and it stays important even when teams add automation.

## What you do as a manual tester

1. **Read requirements** — Understand what the feature should do.
2. **Design test cases** — Decide what to test and in what order.
3. **Execute tests** — Use the app, follow your cases, and note results.
4. **Report bugs** — Log defects clearly so developers can fix them.
5. **Retest** — After fixes, confirm the issue is resolved and nothing else broke.
